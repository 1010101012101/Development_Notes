              Johannes Boonstra Mod Jan-Hendrik Kruger • 15 days ago

              Hello,
              Unlike the HUB4 assistant (which can use an external SOC value from a BMV), the HUB1 does not have a specific capacity reserve is case of grid failure.
              The setting which can limit the depth of discharge based on voltage is load shedding.
              If only you select this (actual wiring of the output isn't needed) the battery discharge is limited by the DC Low Pre alarm. (which for load shedding is the trigger to switch off non-essential loads)
              For 75% that will be roughly 12.6Vdc.

                                          Johannes Boonstra Mod Buckley Davy • 2 months ago

                                          Hi,
                                          There might be a solution for that but I am not 100% sure as I didn't test it myself.
                                          You could try add the assistant "charge current control" and program as in the summary.
                                          Also you need to place a jumper on AUX1 so that is shorted.
                                          Then the policy " keep batteries charged" must be used which prioritizes the battery to be full.
                                          When the generator (On AC input 1) is used it will charge.
                                          When grid (On AC input 2) is active it will not charge, but will passthrough.
                                          The solar will first top-up the batteries and then supply the AC loads /feedback.

                                                                                                                              HUB1 is ment to be used when there is a stabiel grid, when having (very) unreliable grid the system would most likely be better of being set up as off grid system (and with an ignore AC condition or such)
