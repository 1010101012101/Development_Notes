                                                                                          Stephen C Phillips Mod ben • 7 months ago

                                                                                          There is one part in your script which is a bit odd: you've moved the PID file into a subdirectory of /var/run. You've then added a check in the do_start function to create the subdirectory if it doesn't exist. This is all an extra complication and I am not sure why you bother? In doing so you have added a chown command to the new folder but you are changing the ownership to the DAEMON_USER which is probably wrong.

                                                                                          The init script will run as root when the machine boots. When running it manually from the command-line it also need to be run as root (e.g. using sudo). The init script writes the PID file. The DAEMON_USER defined in the init script is the user that the Python script will run as. The choice of DAEMON_USER depends on what the Python script does: making it root will ensure things work but is not ideal from a security perspective.

                                                                                          I suggest just following my init script (remove the PID subdirectory) and see if that helps.

                                                                                                                                              Stephen C Phillips Mod Byron • a year ago

                                                                                                                                              It is written in "shell" script. When you make a text file like this executable and tell the computer to run it, the computer looks at the first line to see if it starts with "#!" (a special comment line) and then uses the rest of the line as the "interpreter" to execute the commands. For instance, Python scripts often start with "#!/usr/bin/python" so the computer uses the program found there to interpret the rest of the script.

                                                                                                                                              If you do "ls -l /bin/sh" then you will see a line ending with "/bin/sh -> dash". This tells us that (if you are running Raspbian like me) that /bin/sh is just a symbolic link to another program called "dash". A long time ago, /bin/sh was an interpreter itself but it has been replaced by other more advanced but backwards compatible shells like dash (in this case) and also commonly "bash".

                                                                                                                                              Shell scripts like this contain commands the same as you'd type in the shell ("the command line"). Google for "shell scripting 101" or something to learn more.
