                                                                              fideloper Mod James • a year ago

                                                                              The article actually recommends not using that command, and instead use the "apt" module.

                                                                              The likely issue is running that command exactly as you see it is that we aren't telling it to use "sudo", which is needed for running an apt-get command. You can use -s flag to have it use sudo, and -K to have it ask you the sudo password, if a password is required.

                                                                              But I'd suggest just moving on and using the "apt" module.
